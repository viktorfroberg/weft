# Agents

weft is agent-agnostic. Claude Code is the seeded preset; any CLI that reads `--add-dir`-style flags and posts status to an HTTP endpoint slots in with a new agent preset row.

## Launching an agent

Tasks auto-spawn the default agent on create — the compose card's prompt becomes the first user turn via the `{prompt}` argv token, so Claude auto-submits it on startup. From an existing task view:

- **⌘L** launches the default preset as a new tab.
- **⌘T** (or the `+` button at the end of the tab strip) opens a picker — Shell or any configured agent preset.
- The **✨ Claude Code** tab shows a spinning `Loader2`, then switches to a green check on clean exit (or a red X on non-zero).
- Scrollback is preserved when you switch tabs. The PTY child is SIGKILLed when the tab closes.
- Agent tabs show a **↻ Reload** icon — kills and respawns the process. A second (or reloaded) agent doesn't replay the user's original prompt; instead it rides on the preset's `bootstrap_prompt_template` so it knows it's joining an in-progress task.

## Preset templates

Presets live in the `agent_presets` SQLite table (seeded at first boot), are editable from **Settings → Agents**, and are resolved at launch time by `src-tauri/src/services/agent_launch.rs`.

A preset has:

| Field | Meaning |
|---|---|
| `id` | Stable identifier. |
| `name` | Shown on the tab + ⌘T picker. |
| `command` | Binary to exec (e.g. `claude`). |
| `args_json` | JSON array of arg templates with `{…}` placeholders. |
| `env_json` | JSON object of env overrides. Merged with weft's injected env. |
| `bootstrap_prompt_template` | Orientation text used when a second agent joins an existing task (prompt already consumed). Nullable. |
| `bootstrap_delivery` | Where the bootstrap template lands in argv — `argv` (portable; uses `{prompt}` slot) or `append_system_prompt` (Claude-only; uses `--append-system-prompt` so Claude doesn't reply to orientation). |

### Placeholders

| Token | Expands to |
|---|---|
| `{slug}` | Task slug (e.g. `pro-2406`). |
| `{branch}` | Full branch name from `tasks.branch_name` (`weft/<slug>` or `feature/<ticket-ids>`). |
| `{primary}` | Path to the first-ready worktree (the agent's cwd). |
| `{each_path:<flag>}` | Repeats `<flag> <path>` for every ready worktree. E.g. `{each_path:--add-dir}` → `--add-dir /path/a --add-dir /path/b`. |
| `{prompt}` | The task's first-turn message (user prompt + linked-ticket summary) for the *very first* agent launch of a task, before `tasks.initial_prompt_consumed_at` is set. Drops silently on every subsequent launch so a relaunch doesn't resubmit a stale user turn. In `argv` delivery mode it also falls through to the bootstrap template; in `append_system_prompt` mode it just drops (bootstrap rides on `{bootstrap}`). |
| `{bootstrap}` | Expands to the preset's `bootstrap_prompt_template` (with `{slug}` / `{branch}` / `{primary}` substituted) when no fresh user prompt is available. Empty → the preceding literal `--flag` is dropped too, so `["--append-system-prompt", "{bootstrap}"]` never strands the flag with no value. |

The seeded Claude Code preset (migration 0009):

```
command:    claude
args_json:  ["--name", "{slug}", "{prompt}", "--append-system-prompt", "{bootstrap}", "{each_path:--add-dir}"]
env_json:   {}
bootstrap_delivery:          append_system_prompt
bootstrap_prompt_template:   "You have joined a weft task in progress. A shared brief for this task is in `.weft/context.md` … Read it before acting, then wait for my instructions."
```

> **Ordering matters.** `--add-dir <dirs...>` is a commander.js variadic — any positional arg AFTER it gets eaten as another directory. Keep `{prompt}` (and any other positional) *before* `{each_path:--add-dir}`. Discovered the hard way; don't re-order.

The launch pipeline:

1. Resolve the preset template against the task's worktrees. If the task still has an unconsumed `initial_prompt`, `services/task_context::compose_first_turn` stitches the prompt + cached Linear ticket titles into the `{prompt}` slot. After a successful spawn the frontend fires `task_consume_initial_prompt`.
2. Write the hook Bearer token into `~/Library/Application Support/weft/hooks.json`.
3. Spawn the PTY with these env vars merged in:
   ```
   WEFT_TASK_ID=<uuid>
   WEFT_TASK_SLUG=<human-slug>
   WEFT_TASK_BRANCH=<branch_name>
   WEFT_HOOKS_URL=http://127.0.0.1:17293/v1/events
   WEFT_HOOKS_TOKEN=<per-launch random string>
   ```
4. Stream stdout through `Channel<Vec<u8>>` to xterm.js.

## Shared task context

Every task's worktrees share a brief at `.weft/context.md`, regenerated by `services/task_context::refresh_task_context` on every mutation (task create, ticket link/unlink, repo add/remove). The file has two fenced sections:

- `<!-- weft:auto-begin -->` … `<!-- weft:auto-end -->` — regenerated from DB (user prompt, linked tickets with cached titles, repo layout). Treat as read-only.
- `<!-- weft:notes-begin -->` … `<!-- weft:notes-end -->` — user AND agent scratch space. Preserved byte-for-byte across auto-regen. Any agent with file access can append findings here so later tabs see them.

A **Claude-specific mirror** lives at `~/.weft/worktrees/<slug>/CLAUDE.md` (above the repo checkouts). Claude's memory loader walks from cwd up to `/`, so this file is auto-discovered alongside any repo-local CLAUDE.md — both concatenated. Non-Claude agents read the per-worktree sidecar instead; the bootstrap template points them at it.

Both paths are gitignored via the common `info/exclude` so `git add -A` won't stage them.

## Auto-rename

After task create, weft runs `claude -p --model haiku --append-system-prompt <prompt-title-instructions>` in the background to replace the heuristic short name with a crisper LLM-generated label. Gated by `prefs.autoRenameTasks` (default `true`) and `WEFT_DISABLE_AUTO_RENAME=1`. A manual rename from the TaskView title sets `tasks.name_locked_at`; any late-arriving LLM rename skips locked rows.

## Hook protocol

weft runs an [axum](https://github.com/tokio-rs/axum) HTTP server on `localhost:17293` (falls back to a next free port; find it via `~/Library/Application Support/weft/hooks.port`).

Agents POST status events so weft can light up sidebar status dots, fire native notifications, and color task-list rows.

### Authentication

Every POST must include `Authorization: Bearer <token>`. The token is:

- Rotated per app launch.
- Written to `~/Library/Application Support/weft/hooks.json`.
- Injected into every terminal weft spawns as `WEFT_HOOKS_TOKEN`.

Requests without the correct token get `401`. Token compromise scope: until next weft restart.

### Event shape

`POST $WEFT_HOOKS_URL` with `Content-Type: application/json`:

```json
{
  "source": "claude_code",
  "task_id": "<value of $WEFT_TASK_ID>",
  "kind": "active" | "waiting_input" | "error" | "session_end",
  "detail": { "anything": "you want, stored for later" },
  "timestamp": 1713357600
}
```

| `kind` | UI effect |
|---|---|
| `active` | Sidebar dot emerald; task row labeled "working". |
| `waiting_input` | Sidebar dot amber; native notification fires (once per enter-state). |
| `error` | Sidebar dot red. |
| `session_end` | Dot subdued; tab status badge flips to final state on PTY exit. |

Any `detail` payload you send is stored alongside the event in the `StatusStore`; not surfaced in v1 but available for later.

### Appending notes to `.weft/context.md`

Agents that want to drop structured notes in the shared brief should post to `/v1/task_context_append` instead of editing the file directly — hand-edits race the fence splicer. weft handles locking, timestamps the entry, and slots it under a `## Agent notes` subsection in the `weft:notes` block of the primary worktree's sidecar.

```bash
curl -sX POST "http://127.0.0.1:$(cat ~/Library/Application\ Support/weft/hooks.port)/v1/task_context_append" \
  -H "Authorization: Bearer $WEFT_HOOKS_TOKEN" \
  -H 'Content-Type: application/json' \
  -d "{\"task_id\":\"$WEFT_TASK_ID\",\"note\":\"Tried approach X, couldn't reproduce the bug without a real DB.\"}"
```

- Payload: `{ "task_id": "...", "note": "markdown string" }`. Bearer is strict here — no empty-token dev bypass.
- Max note length: 4 KiB (413 past that).
- Responses: `200 { appended: true }`, `401` bad token, `404` unknown task, `409` no ready worktrees.
- Write target: primary (first-ready) worktree only. Peer worktrees converge on the next ContextDialog save.

## Wiring Claude Code's native hook system

Claude Code has its own tool/session hooks (see the [Claude Code docs](https://docs.claude.com/claude-code/hooks)). To forward them to weft, configure your `~/.claude/settings.json` (or project-local `.claude/settings.json`) to `curl` the events:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [{
          "type": "command",
          "command": "curl -sX POST \"$WEFT_HOOKS_URL\" -H \"Authorization: Bearer $WEFT_HOOKS_TOKEN\" -H 'Content-Type: application/json' -d '{\"source\":\"claude_code\",\"task_id\":\"'$WEFT_TASK_ID'\",\"kind\":\"active\",\"timestamp\":'$(date +%s)'}'"
        }]
      }
    ],
    "Stop": [
      {
        "matcher": "",
        "hooks": [{
          "type": "command",
          "command": "curl -sX POST \"$WEFT_HOOKS_URL\" -H \"Authorization: Bearer $WEFT_HOOKS_TOKEN\" -H 'Content-Type: application/json' -d '{\"source\":\"claude_code\",\"task_id\":\"'$WEFT_TASK_ID'\",\"kind\":\"waiting_input\",\"timestamp\":'$(date +%s)'}'"
        }]
      }
    ]
  }
}
```

The env vars are already in the PTY. `curl -s` keeps noise off your terminal output.

## Adding a new agent

Open **Settings → Agents → New preset**. Fill in:

- **Name** — label shown in the ⌘T picker and Launch menu (e.g. "Codex").
- **Command** — the binary, e.g. `codex`.
- **Args** — a JSON array of template strings. Use the placeholders listed above (`{slug}`, `{prompt}`, `{each_path:<flag>}`, `{bootstrap}`, …).
- **Env** — a JSON object of string→string env vars. Values are masked in the dialog by default; click **Reveal** to edit.
- **Bootstrap prompt template** (optional) — orientation text for second-agent launches, when the task's `initial_prompt` is already consumed.
- **Bootstrap delivery** — `argv` (portable, slots into `{bootstrap}`) or `append_system_prompt` (Claude-only, keeps the orientation out of the visible transcript).

Save. No restart required — the preset shows up in the ⌘T picker and Launch menu immediately. Set it as default from the same row if you want new tasks to pick it up automatically. Write your own agent-flavored hooks using the same token → endpoint pattern.
